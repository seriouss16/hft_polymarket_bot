# История Git после выноса

## Метод

1. Клон монорепозитория `prjBJ_arb_polymarket` во временный каталог.
2. `git filter-repo --subdirectory-filter hft_bot` — в корне репозитория только содержимое бывшего `hft_bot/`, история сохранена для всех коммитов, затрагивавших эту папку.
3. Результат перенесён в `/mnt/work/DEV/PRJ0/prjBH_hft_bot`, поверх добавлены `pyproject.toml`, `uv.lock`, `.gitignore`, точка входа `hft-bot` (`bot.run_cli`).

## Соответствие коммитов

| Монорепозиторий | Этот репозиторий |
|-----------------|------------------|

Проверка: `git rev-parse bd7ac4f:hft_bot` в монорепо должна совпадать с `git rev-parse 3638c01^{tree}` здесь.

## Что не делали

- **Не** обрезали историю командой вида `git checkout bd7ac4f && git branch main` — это удалило бы все последующие коммиты. Актуальное состояние — кончик ветки после filter-repo (аналог `main` монорепо по части бота).
- Ветка **`main_short`** в монорепозитории — указатель на **initial commit** (`6c32aad…`), для ориентира по состоянию репозитория до появления `hft_bot`.

## Пустой корневой коммит

После `filter-repo` первый коммит в линии может иметь пустое дерево (наследие общего корня монорепо). Это нормально для `git filter-repo`; рабочее дерево появляется с коммитами вроде `3638c01`.
